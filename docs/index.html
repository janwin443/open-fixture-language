Why OFL

Designed for lighting,
not for computers.

Every other fixture format was built around data structure. OFL is built around how lighting people actually think.

⚡
Zero Dependencies

Pure Python 3.10+. No pip install. No bloat. The entire parser fits in your project folder.

🔗
@ Includes

Share channel definitions across fixtures. Local definitions always override imported ones.

🎯
16bit Native

Pan and tilt in two channels? Write 1,2: pan in your mode. Coarse and fine are implicit.

🔄
Full Roundtrip

Load a fixture, modify it in Python, write it back. The output is identical to the input.

🛡
Validated

The Validator catches invalid attributes, cap range gaps, and undefined channel references before runtime.

🧩
Embeddable

Six files. Drop them into any Python project. Works standalone or as part of a larger lighting stack.

Syntax

Readable as a manual.
Writable as code.

One file per fixture. No arrays. No nesting hell. Just fixture definitions that look exactly like what they describe.

robe_robin600e.ofl OFL 
# Robe Robin 600E Spot

fixture Robin600E:
    manufacturer: Robe
    type: moving_head
    weight: 19.5kg
    power: 480W

    @shared/pan_tilt_16bit        # pulls in pan + tilt channels

    channel dimmer:
        attribute: intensity
        resolution: 16bit
        cap 0..65535: open

    channel gobo:
        attribute: gobo
        resolution: 8bit
        cap 0..9:   open
        cap 10..19: gobo(slot=1)
        cap 20..29: gobo(slot=1, spin=cw)

    channel color:
        attribute: color
        resolution: 8bit
        cap 0..9:   open
        cap 10..19: color(hex=#FF0000, name=Red)
        cap 20..29: color(hex=#0000FF, name=Blue)

    mode standard(16ch):
        1,2: pan       # 16bit — coarse + fine
        3,4: tilt
        5,6: dimmer
        7:   gobo
        8:   color

    mode basic(8ch):
        1: pan
        2: tilt
        3: dimmer
        4: gobo

Usage

Drop in. Parse. Done.

No configuration. No class registration. Just import and run.

load a fixture 
from pathlib import Path
from lexer     import Lexer
from parser    import Parser
from resolver  import Resolver
from validator import Validator
from builder   import Builder

source   = Path("robin600e.ofl").read_text()
tokens   = Lexer(source).tokenize()
ast      = Parser(tokens).parse()
resolved = Resolver(Path("fixtures/"))\
             .resolve(ast, Path("robin600e.ofl"))
Validator().validate(resolved)
fixture  = Builder.build(resolved)

# done.
print(fixture.name)
print(fixture.channels)
write back 
from writer import Writer

# modify the fixture
fixture.manufacturer = "Custom"

# write back to .ofl
text = Writer.write(fixture)
Path("output.ofl").write_text(
    text, encoding="utf-8"
)

# output is identical to input
# — full roundtrip guaranteed.
